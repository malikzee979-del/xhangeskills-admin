BC220408949 Table of Contents                         Introduction of Design Document Entity Relationship Diagram (ERD) Sequence Diagrams Architecture Design Diagram Class Diagram Database Design Interface Design Test Cases      Introduction of Design Document This Software Design Document presents the detailed design of the  XchangeSkills platform . The purpose of this document is to describe how the requirements specified in the Software Requirements Specification (SRS) are translated into a concrete system design prior to implementation. The document explains the overall system architecture, data design, and key components involved in the platform. It provides a technical blueprint that outlines how different parts of the system interact to support functionalities such as user management, skill listing, service requests, booking management, chat communication, and reviews. This design document follows the  VU Process Model , ensuring that the system design is developed after requirement finalization and before implementation. While the SRS defines  what  the system should do from a user perspective, this document focuses on  how  the system will internally manage data, interactions, and control mechanisms to ensure reliability, scalability, and fairness. The XchangeSkills platform is designed using a  JAMstack-based architecture , where  Next.js  is used for frontend development and  Strapi  is used as a headless backend that exposes functionality through RESTful APIs. This design approach supports modular development, ease of maintenance, and future scalability. Overall, this document serves as a reference for developers, evaluators, and stakeholders to understand the internal design decisions of the system and to ensure that the final implementation remains consistent with the approved requirements. Entity Relationship Diagram (ERD) (To be developed using Microsoft Visio or any other drawing software of your choice) Overview The Entity Relationship Diagram (ERD) represents the logical data model of the  XchangeSkills platform . It identifies the main entities involved in the system, their attributes, and the relationships between them. The ERD provides a clear understanding of how data is structured and how different entities are linked to support the systemâ€™s core functionalities. The ERD has been designed with a focus on  data normalization, integrity, and scalability . Separate entities are defined for users, roles, profiles, skills, categories, booking requests, chat sessions, messages, and reviews to avoid redundancy and to support efficient data management. The diagram illustrates how users can offer skills, request services, participate in chats, and provide reviews after service completion. It also reflects the systemâ€™s ability to manage service requests with defined scope, time, duration, and location, ensuring proper coordination between service providers and requesters. The ERD is developed using  Microsoft Visio or an equivalent diagramming tool , and it serves as the foundation for database design and implementation in later phases of the project.
