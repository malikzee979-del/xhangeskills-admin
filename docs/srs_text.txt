Many people possess valuable abilitiesâ€”such as web design, tutoring, photography, cooking, or household skillsâ€”but cannot always afford to pay for services they need. This project addresses that gap by enabling a structured, fair skill-bartering system within the community. The scope of the system includes user registration, profile creation, and the ability to list offered skills as well as request needed skills. Users will be able to browse, search, and filter skills by category, location, and keywords. The platform will support a booking/request mechanism that allows two parties to arrange a skill exchange. Email-based notifications will guide users through confirmations, approvals, reminders, and completion updates. After each exchange, users will be able to provide ratings and reviews to maintain service quality and trust. To make communication easier and reduce dependency on external apps, the system will include a simple in-app chat feature. Once a booking request is created, both users involved can exchange text messages to clarify requirements, discuss timing, and finalize details of their skill exchange. The chat will be lightweight, text-only, and designed purely for coordination, without file sharing or audio/video features. Technologically, the system will be built using Next.js 15 and React 19 for the frontend, Strapi v5 as the backend CMS, and PostgreSQL (or MongoDB) as the database. JWT-based authentication will secure the platform, while Cloudinary will handle media storage and the Strapi Email Plugin will manage email notifications. This project focuses on delivering a practical, usable, and scalable skill-exchange platform with essential bartering features, ensuring safe, smooth, and meaningful interactions among community members. Functional and non Functional Requirements: Functional Requirements: User Registration and Authentication The system shall allow users to sign up using basic information. The system shall allow users to log in using secure authentication. The system shall allow users to update their profile information. The system shall allow users to reset their password via a â€œForgot Passwordâ€ feature. The system shall allow authenticated users to change their password. Skill Management Users shall be able to add new skills they offer. Users shall be able to edit or delete their posted skills. Each skill shall include a title, description, category, and optional image. Skill Browsing and Search The system shall allow users to browse all available skills. The system shall allow users to search skills using keywords, categories, or location filters. The system shall display detailed information for each skill. Booking and Request Management Users shall be able to send a booking request for a skill. Users shall be able to view their sent and received requests. The receiver shall be able to accept or reject the request. The system shall update the status of each request accordingly. Email Notifications The system shall send email notifications for account creation, booking requests, request updates, and reminders. In-App Chat The system shall provide a simple text-based chat once a booking request is created. Users shall be able to exchange messages to coordinate session details. Chat shall be limited to text-only communication. Ratings and Reviews After completing an exchange, users shall be able to rate and review each other. The system shall display average ratings on user profiles and skill listings. Non Functional Requirements: Usability The system shall provide a clean, intuitive interface that is easy to navigate. Performance Pages shall load within a reasonable time (2â€“3 seconds). The system shall handle typical user traffic efficiently. Security Passwords shall be securely hashed. Sensitive data and user communication shall be transmitted securely. All inputs shall be validated to prevent common attacks. Reliability Core features shall work consistently without unexpected failure. User data shall be stored safely and remain available. Scalability The system shall be capable of handling growth in users, skills, and chat messages without major performance loss. Adopted  Methodology The methodology adopted for this project is the VU Process Model, which is a hybrid approach combining the structured nature of the Waterfall Model with the iterative flexibility of the Spiral Model. This model is recommended by Virtual University for software projects because it provides both planning discipline and controlled risk handling. The Waterfall component ensures a clear sequence of stages, including requirements gathering, system design, development, testing, deployment, and documentation. Each phase is completed with well-defined deliverables, making it easier to track progress and maintain high-quality standards throughout the project. The Spiral component adds iterative cycles that allow continuous evaluation, feedback incorporation, and risk management. After completing the major Waterfall phases, the team reviews progress, identifies risks, and refines requirements, design, or implementation where needed. This approach reduces the chances of critical design flaws and ensures that the system evolves according to user expectations and project constraints. Using the VU Process Model for this Skill Exchange Platform allows the team to first establish solid foundational requirements and architecture, then iteratively refine features such as skill posting, booking requests, chat functionality, and user experience. With each cycle, improvements can be made to usability, security, and performance before proceeding to the final deployment. This combination of structured planning and iterative refinement makes the VU Process Model an ideal choice for developing a scalable, reliable, and user-centered application. This project also adopts the  JAMstack architectural approach . The frontend is developed using  Next.js , which handles the user interface and client-side logic, while the backend is implemented using  Strapi , a headless CMS that provides data and services through RESTful APIs. This separation of frontend and backend aligns with JAMstack principles and supports scalability, performance, and iterative development within the VU Process Model.
